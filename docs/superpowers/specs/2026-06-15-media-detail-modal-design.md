# Медиа: модалка-превью по клику (полное изображение, инфо, alt, действия)

- **Дата:** 2026-06-15
- **Issue:** [#72](https://github.com/alshchetinin/zhk-starter/issues/72)
- **Ветка:** `feat/72-media-detail-modal`

## Проблема

В разделе «Медиа» (`apps/admin/app/pages/media/index.vue`) карточка показывает
превью `object-cover` (обрезанное), имя, размер, бейдж папки и **инлайн-поле alt**.
Нельзя посмотреть изображение целиком и его метаданные, а редактирование alt
зажато в маленький `UInput` прямо в сетке — это шумит в карточке и не даёт
контекста (не видно самой картинки в полном виде при правке описания).

## Решение

Клик по превью открывает модалку с полным изображением, метаданными и
редактированием alt. Инлайн-поле alt из карточки убираем (редактирование alt —
только в модалке). Бэкенд не трогаем: `media.update` (alt по `url`) и
`media.delete` уже есть; копирование URL, скачивание и размеры изображения —
на клиенте.

### 1. Новый компонент `apps/admin/app/components/MediaDetailModal.vue`

На `UModal` (см. правило проекта «UI в admin: только @nuxt/ui»). Раскладка —
две колонки на десктопе, стек на мобиле.

**Пропсы / модель:**

```ts
defineProps<{ items: MediaItem[] }>();   // текущая страница списка (для листания)
const index = defineModel<number | null>("index"); // null → закрыто
```

`open` = `computed(() => index.value !== null)`; текущий элемент —
`items[index.value]`. Закрытие — `index.value = null`.

**Левая колонка** — полное изображение `object-contain` на нейтральном фоне
(`--ui-bg-elevated`/`tertiary`). Стрелки ←/→ (кнопки + хоткеи клавиатуры),
листание **в пределах `items`** (текущая страница, 24 шт.); на краях кнопки
`disabled`. Esc закрывает (дефолт `UModal`).

**Правая колонка** — инфо + alt + действия:
- Инфо-таблица: тип (`contentType`), размер файла (`formatFileSize`), **размеры
  Ш×В** (читаются из `naturalWidth/naturalHeight` по `@load` картинки, в БД их
  нет), папка (`folder`), дата загрузки (`createdAt`), URL (усечённый) с кнопкой
  «копировать» (`navigator.clipboard` + тост).
- Поле **alt**: `UTextarea`, `maxlength=500` (ограничение схемы `media.update`),
  автосохранение на blur — оптимистично (см. ниже).
- Кнопки: «Скачать» (ссылка на `url` с `download`) и «Удалить» (с подтверждением
  → `media.delete`).

### 2. Изменения `apps/admin/app/pages/media/index.vue`

- Убрать инлайн `UInput` alt и логику `altDrafts` / `commitAlt` из карточки.
- Превью (`<img>`) обернуть в `<button>` → `selectedIndex.value = i` (открыть модалку).
- Карточкам **без alt** добавить маленький бейдж «без alt» (видимость для
  SEO-ревизии — компенсирует убранное инлайн-поле).
- Состояние: `const selectedIndex = ref<number | null>(null)`; рендер
  `<MediaDetailModal :items="items" v-model:index="selectedIndex" />`.
- Удаление: модалка переиспользует ту же `deleteMutation`; после успеха —
  закрыть модалку и скорректировать индекс (если удалили последний на странице).

### 3. Синхронизация alt (оптимистично)

Правка alt в модалке и сетка должны быть в синхроне без мигания. На blur:
`setQueriesData({ queryKey: $orpc.media.list.key() })` — обновить `alt` элемента
в кеше списка, затем `media.update`, в `onError` откат из снапшота, в `onSettled`
`invalidateQueries`. Паттерн — как для detail-страниц в `CLAUDE.md`
(«Admin data-fetching»).

## Границы (YAGNI)

- Листание ←/→ — **только в пределах текущей страницы** (24). Подгрузка соседних
  страниц списка в модалке не делается.
- Модалку открывают только изображения; карточки не-изображений
  (`contentType` не `image/*`) поведение не меняют.
- Зум/панорамирование изображения не делаем — только `object-contain` по размеру.

## Проверка

`pnpm dev` (admin :3002) → раздел «Медиа»:
1. Клик по превью → открывается модалка с полным изображением и метаданными.
2. Размеры Ш×В отображаются после загрузки картинки.
3. Правка alt → после перезагрузки страницы сохранена; сетка (бейдж «без alt»)
   обновляется.
4. Листание стрелками и клавишами ←/→; на краях страницы кнопки неактивны.
5. Копирование URL (тост), скачивание файла, удаление (с подтверждением →
   модалка закрывается, файл исчезает из сетки).
6. Адаптив: на узком экране колонки в стек.
